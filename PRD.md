# TermScope PRD（简版）

## 项目名
暂定：**TermScope**

## 项目类型
基于 **Python + Textual** 的 Linux/服务器监控 TUI 仪表盘。

## 一句话描述
在终端中实时展示 Linux 主机的关键运行指标，包括 **发行版信息、CPU、内存、磁盘、网络、负载和进程概览**。

---

## 背景与目标
开发者和运维经常需要快速查看服务器状态，但：

- `top` / `htop` 偏单一
- `neofetch` 只展示静态系统信息
- 多个命令切换查看效率低

本项目希望提供一个：

- **信息集中**
- **键盘友好**
- **实时刷新**
- **具备“主机身份 + 资源状态 + 进程概览”统一视图**

的终端仪表盘。

---

## 目标用户

- Linux 开发者
- 个人服务器用户
- 自托管爱好者
- 运维 / DevOps 初学者
- 喜欢终端工具的程序员

---

## 核心使用场景

### 场景 1：快速确认当前机器
用户打开程序后，立即看到：

- 主机名
- 发行版
- 版本
- 内核
- uptime

### 场景 2：检查机器资源健康度
用户快速查看：

- CPU 使用率
- 内存使用率
- 磁盘占用
- load average
- 网络收发速率

### 场景 3：发现占用资源最多的进程
用户查看：

- Top CPU 进程
- Top Memory 进程

## MVP 目标
首版只解决一个核心问题：

> “让我在一个 TUI 界面里，快速了解当前 Linux 主机是谁、状态怎么样、谁最占资源。”

---

## MVP 功能需求

### 1. 系统身份信息
展示以下内容：

- hostname
- Linux 发行版名称
- 发行版版本
- pretty name
- kernel version
- architecture
- uptime

**数据来源：**

- `/etc/os-release`
- `platform.uname()`
- `socket.gethostname()`
- `/proc/uptime`

### 2. 实时系统指标
展示以下内容：

- CPU 使用率
- 内存使用率
- Swap 使用率
- 根分区磁盘使用率
- Load average (1/5/15)
- 网络接收速率
- 网络发送速率

**数据来源：**

- `psutil`
- `os.getloadavg()`

### 3. 进程概览
展示：

- CPU 占用前 N 个进程
- 内存占用前 N 个进程

每个进程显示：

- PID
- name
- user
- CPU%
- MEM%

**数据来源：**

- `psutil.process_iter()`

### 4. TUI 页面
MVP 至少包含 2 个页面：

#### Dashboard

- 顶部：主机身份信息
- 左侧：系统指标
- 右侧：Top 进程
- 底部：快捷键提示 / 状态栏

#### Processes

- 更完整的进程表格
- 支持按 CPU / MEM 排序切换

### 5. 自动刷新

- 默认每 1 秒刷新一次
- 支持手动刷新

### 6. 键盘交互

- `q`：退出
- `r`：手动刷新
- `d`：Dashboard
- `p`：Processes

## 非目标
MVP 暂不做：

- 多主机监控
- SSH 远程采集
- Docker / 容器监控
- systemd 服务状态
- 历史数据库持久化
- 告警规则编辑
- 图形化复杂折线图

---

## 成功标准
MVP 成功的标准：

1. 在主流 Linux 发行版上可运行
2. 能正确识别并显示发行版信息
3. 指标每秒更新，无明显卡顿
4. 能稳定展示前几名高占用进程
5. 界面清晰、键盘操作自然

## 风险与注意事项

### 1. 发行版兼容性
不是所有系统的 `/etc/os-release` 字段都完全一致，需要容错。

### 2. 网络速率计算
`psutil.net_io_counters()` 是累计值，需要做差分才能算实时速率。

### 3. 进程 CPU%
首次采样值可能不准确，需要预热或二次采样。

### 4. 容器 / 极简系统
某些环境可能没有完整的 `/etc/os-release` 或 load average 信息，要降级处理。
